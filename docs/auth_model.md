# Модель авторизации

## Регистрация

1. Пользователь передает `email`, `username`, `password` и выбранную роль.
2. Сервер проверяет уникальность email и username.
3. Пароль хешируется функцией bcrypt.
4. В таблицу `users` сохраняется только `password_hash`.

Открытый пароль в базе не хранится.

## Вход

1. Пользователь передает email и пароль.
2. Сервер находит активного пользователя по email.
3. Пароль сравнивается с bcrypt-хешем.
4. При успешной проверке сервер выдает JWT.

## JWT

JWT содержит:

- `sub` - идентификатор пользователя.
- `exp` - срок действия токена.

Токен передается в заголовке:

```text
Authorization: Bearer <token>
```

## Проверка доступа

- Для чтения чата пользователь должен быть участником `chat_members`.
- Для отправки сообщения нужна глобальная роль `writer` или `admin`.
- Роль `reader` может отправлять сообщения только в личный чат «Избранное».
- Участник с ролью `readonly` может читать чат, но не может отправлять сообщения.
- `admin` может просматривать список пользователей.

## Аудит

События регистрации, входа, создания чата, отправки сообщений, реакций и вложений пишутся в `audit_logs`.
