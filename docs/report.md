# Отчет по практике: кейс №1 «Мессенджер»

## Предметная область

Проект реализует сервис быстрого обмена сообщениями. Пользователь регистрируется, входит в систему, создает личные и групповые чаты, отправляет и получает текстовые сообщения, прикрепляет файлы к сообщениям, ищет сообщения по тексту и сохраняет важные сообщения в «Избранное». Данные хранятся в PostgreSQL, а загруженные файлы сохраняются в каталоге приложения `app/uploads`, при этом в базе данных фиксируются их метаданные.

## Требования MVP из задания

| Требование | Реализация |
|---|---|
| Сервер и клиент мессенджера | FastAPI-сервер, Web UI в `app/static/index.html`, CLI-клиент в `client/cli.py` |
| Авторизация пользователя | JWT, bcrypt-хеши паролей, эндпоинты `/api/auth/register` и `/api/auth/login` |
| Чаты | Таблицы `chats`, `chat_members`, эндпоинты `/api/chats` |
| Быстрая отправка и получение сообщений | REST-отправка и WebSocket `/ws/chats/{chat_id}` |
| Поиск по сообщениям | `/api/messages/search`, PostgreSQL `pg_trgm`, индекс `ix_messages_search` |
| Отправка файлов | `POST /api/uploads`, таблица `attachments`, отправка файлов вместе с текстовым сообщением |

## Дополнительные возможности

- UI-клиент.
- Статусы сообщений: `sent`, `delivered`, `read`.
- Реакции.
- Файловые сообщения: до 3 файлов в одном сообщении, предпросмотр перед отправкой, удаление выбранных файлов до отправки, открытие файлов в новой вкладке.
- Текстовая подпись к файловому сообщению с ограничением до 200 слов.
- Добавление файловых сообщений в «Избранное» с сохранением ссылки на исходный чат и исходное сообщение.
- Ограничение частоты отправки сообщений.
- Аудит действий пользователей.

## Архитектура

Система состоит из:

- `api` - FastAPI-приложение.
- `db` - PostgreSQL.
- `web client` - статический HTML/CSS/JS клиент.
- `cli client` - консольный клиент на Python.
- `uploads` - каталог `app/uploads` для хранения загруженных пользовательских файлов.

## Структура базы данных

Основные сущности:

- `users` - учетные записи и роли.
- `chats` - чаты.
- `chat_members` - связь пользователей с чатами.
- `messages` - сообщения.
- `reactions` - реакции на сообщения.
- `attachments` - метаданные файлов, прикрепленных к сообщениям.
- `audit_logs` - аудит действий.

ER-диаграмма приведена в `docs/er_diagram.md`.

## API

API описан в `docs/api.md`. Интерактивная документация доступна после запуска по адресу `/docs`.

Для файловых сообщений используется отдельный сценарий: сначала файл передается на сервер через `POST /api/uploads`, затем полученные `file_name`, `file_url` и `mime_type` отправляются вместе с сообщением через `POST /api/chats/{chat_id}/messages`. Такой подход отделяет загрузку бинарных данных от создания записи сообщения и позволяет хранить в базе только необходимые метаданные.

## Авторизация и безопасность

Модель авторизации описана в `docs/auth_model.md`.

Ключевые решения:

- Пароли хранятся только в виде bcrypt-хешей.
- После входа пользователь получает JWT.
- Доступ к чатам проверяется через таблицу `chat_members`.
- Пользователь не видит сообщения чужих чатов.
- Отправка файлов подчиняется тем же правилам доступа, что и отправка обычных сообщений.
- Сервер проверяет ограничение до 3 файлов и до 200 слов подписи к файловому сообщению.
- Действия пишутся в `audit_logs`.

## Файловые сообщения

Пользователь может прикрепить файл через кнопку со скрепкой в строке набора сообщения. Перед отправкой выбранные файлы отображаются в панели предпросмотра: показываются название файла и расширение, а каждый файл можно удалить отдельно через крестик. Это позволяет отказаться от отправки конкретного файла до нажатия кнопки отправки.

Файл можно отправить отдельно или вместе с текстовой подписью. Если вместе с файлами добавлена подпись, она ограничивается 200 словами. При превышении лимитов интерфейс показывает всплывающее уведомление в стиле приложения.

В чате файловое сообщение отображается как единая карточка: сначала выводятся файлы, ниже - текстовая подпись, если она есть. Файл открывается в новой вкладке браузера. Файловые сообщения поддерживают основные сценарии мессенджера: на них можно ответить, их можно удалить, добавить в «Избранное» и перейти из «Избранного» к исходному сообщению в исходном чате.
